# Product

## Register

product

## Users

Codex Port is for developers who want to use their existing Codex CLI setup from an iPhone-first native app. They are usually in a project context and need fast access to host profiles, workspaces, sessions, transcripts, approvals, and file attachments without operating a terminal UI.

## Product Purpose

Codex Port connects an iPhone app to a user's own Mac, VPS, or Linux host, reusing the host's Codex CLI state, project filesystem, credentials, and local setup. Success means a user can reliably resume or start project sessions, send prompts, observe live Codex output, and recover from foreground or network changes with clear state.

## Brand Personality

Quiet, native, work-focused. The app should feel like a dependable iOS tool for repeated technical work rather than a marketing surface or novelty chat client.

## Anti-references

Do not make it feel like a generic ChatGPT clone, SSH terminal, marketing landing page, decorative dashboard, or over-styled AI demo. Avoid visual flourishes that distract from session scanning, connection state, and prompt workflow.

## Design Principles

- Native affordances first: use standard iOS interaction patterns for navigation, swipe actions, buttons, loading, and destructive or archival actions.
- Sessions are work objects: lists should make project, recency, status, and title easy to scan.
- Connection state must be visible but restrained: surface meaningful status without crowding the primary session workflow.
- Optimize for repeated use: row hit targets, grouping controls, and creation actions should be fast and forgiving.
- Keep data provenance clear: remote Codex state is the source of truth, with local optimistic state only as a temporary bridge.

## Accessibility & Inclusion

Use Dynamic Type-compatible text where practical, maintain readable contrast, preserve full-row touch targets for list rows, and prefer standard VoiceOver labels for navigation and swipe actions.
